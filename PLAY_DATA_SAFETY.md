# Google Play "데이터 안전(Data safety)" 폼 답변 초안

> Play Console → 앱 콘텐츠 → 데이터 안전 섹션에 그대로 입력. 실제 동작 기준으로 작성됨.
> (크래시 트래커 Sentry/Crashlytics 미도입 상태 기준 — 도입하면 "앱 활동/진단" 추가 필요)

## 1. 데이터 수집·공유 여부
- 앱이 사용자 데이터를 **수집함**: **예**
- 앱이 사용자 데이터를 **제3자와 공유함**: **아니요**
- 모든 전송 데이터 **암호화(전송 중)**: **예** (HTTPS)
- 사용자가 **데이터 삭제 요청** 가능: **예** (앱 내 회원 탈퇴 → 전체 삭제)

## 2. 수집하는 데이터 유형

### 개인 정보 > 이메일 주소
- 수집함 / 공유 안 함
- 목적: **계정 관리**, **앱 기능**
- 필수(사용자가 끌 수 없음)
- 전송 중 암호화: 예 / 삭제 요청 가능: 예

### 금융 정보 > 기타 금융 정보
(사용자가 직접 입력하는 거래 내역·금액·예산·계좌/카드 메타 — 실제 카드번호 아님)
- 수집함 / 공유 안 함
- 목적: **앱 기능**
- 필수
- 전송 중 암호화: 예 / 삭제 요청 가능: 예

> ※ "사용자 결제 정보 / 신용카드 번호"는 **수집 안 함** — 카드 번호는 저장하지 않고 카드 이름·결제일만 사용.

## 3. 수집하지 않는 항목 (체크 안 함)
- 위치, 연락처, 사진/동영상, 정확한 신원(주민번호 등), 기기 ID 광고용, 메시지 등 — 해당 없음
- 앱 활동/진단(크래시 로그) — 현재 트래커 미도입이라 수집 안 함 (도입 시 갱신)

## 4. 보안 섹션 요약
- 전송 중 암호화: 예 (HTTPS / TLS)
- 저장 데이터: 행 수준 보안(RLS)으로 본인 데이터만 접근, 비밀번호 단방향 해시
- 데이터 삭제: 설정 > 계정 관리 > 회원 탈퇴 → 모든 데이터 연쇄 삭제
