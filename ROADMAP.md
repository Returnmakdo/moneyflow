# 가계부 로드맵

## 입금 기능 (지출 → 전체 흐름 관리로 확장)

가계부 앱이 "지출 관리"에서 "수입·지출·잔액 흐름 관리"로 확장되는 큰 변화.
Phase 단위로 잘게 쪼개서 점진적으로.

### Phase 1. 데이터 모델 + 기본 입력 (필수, 작업 중간)
**목표**: 입금 거래를 등록·표시·집계할 수 있다.
- `transactions`에 `type` 컬럼 추가 (`income` | `expense`)
- 거래 추가 모달에 수입/지출 토글
- 카테고리에 type 추가 (수입 카테고리 별도: 월급/이자/용돈/기타수입)
- 거래내역 row 색상 분기 (수입 파랑/+, 지출 빨강/-)
- 거래내역 필터에 수입/지출 세그먼트
- API/모델 전반에 type 처리

### Phase 2. 대시보드 통합 (Phase 1 후, 작업 중간)
**목표**: 한 달 흐름이 한눈에.
- KPI: "이번 달 지출" → "지출 · 수입 · 흑자" 3분할
- 6개월 추이 막대 → 수입·지출 분리 (또는 흑자 라인)
- "저축률 N%" 카드 추가

### Phase 3. AI 분석 확장 (Phase 1·2 후, 작업 작음)
**목표**: 저축률·소득 대비 지출 비율 짚어준다.
- 분석 결과에 "흑자/적자 · 저축률" 섹션
- 큰 수입(보너스 등)도 이상치로 표시
- spending-insights Edge Function prompt 확장

### Phase 4. 정기 거래 일반화 (Phase 1 후, 작업 작음)
**목표**: 월급 같은 정기 수입도 자동.
- "정기지출" → "정기 거래"로 명칭/모델 일반화
- 정기 거래에 type 추가
- 일괄 등록 시 두 종류 다 처리

### Phase 5. 목표 시스템 (Phase 1·2 후, 작업 중간)
**목표**: 사용자가 정한 저축·지출 목표 추적.
- "이번 달 50만원 저축" 같은 저축 목표 설정 화면
- 카테고리별 지출 한도 (기존 예산 확장)
- 달성률 카드

### Phase 6. 은행 거래내역 import (Phase 1 후, 작업 작음)
**목표**: 입출금 분리된 은행 양식도 자동 인식.
- AI 매핑 schema에 type 추가
- 카카오뱅크/토스뱅크/시중은행 거래내역 양식 자동 인식
- 현재 미지원 → 지원

### Phase 7. 잔액 추적 (옵션, 작업 큼)
**목표**: 통장형 가계부.
- 시작 잔액 입력 + 거래마다 누적
- 대시보드에 잔액 카드
- 통장 앱 수준의 변화

**추천 진행 순서**: 1 → 2 → 3 → 6 → 4 → 5. 7은 별도 결정.
